## 1. Executive answer

The post-`073` symmetry picture is now:

- **symmetric:** the global `beta` clock, hence the carry rotation;
- **not symmetric:** the splice update that chooses the next decorated block;
- **correct interpretation of the remaining asymmetry:** not a static chain
  label, not a parity-like invariant of the current chain, but a **future-word
  / right-congruence state** attached at the boundary;
- **right final claim:** the exact global bridge should be stated as
  `(beta, delta)` where `delta` is the canonical boundary right-congruence
  state of the future decorated-block word.

So the answer is neither

- “bare `beta` is exact,” nor
- “carry remains intrinsically asymmetric across chains.”

The sharper answer is:

> **carry is globally rotatable, and the only surviving asymmetry is dynamic
> splice memory carried on top of that rotatable clock.**

The main remaining open point is quantitative:

> does that dynamic memory compress to a small uniform decorated state, or can
> it grow with the splice orbit?

## 2. The right setup: blocks, not raw chains

After `072–073`, the natural unit is a full regular block from `carry_jump` to
`wrap`.

Write the successive regular blocks on one accessible component as

\[
C_0, C_1, C_2, \dots
\]

with block coordinate

\[
\beta \in \{m-1,m-2,\dots,1,0\},
\]

where `beta = m-1` at `carry_jump` and `beta = 0` at `wrap`.

Each block has a static two-bit **block type**

\[
t_n = (a_n,b_n) \in \{0,1\}^2,
\]

and therefore one of the four length-`m` event words

```text
W_00 = carry_jump, flat,          flat,          ..., flat, wrap
W_01 = carry_jump, flat,          other_0010,    ..., flat, wrap
W_10 = carry_jump, other_1000,    flat,          ..., flat, wrap
W_11 = carry_jump, other_1000,    other_0010,    ..., flat, wrap
```

The full current-event word on the component is the concatenation

\[
W(t_0)\,W(t_1)\,W(t_2)\cdots .
\]

This formulation isolates the symmetry cleanly:

- the within-block transport is the same on every block;
- all nontrivial variation lives in the boundary sequence
  `t_0, t_1, t_2, ...`.

That is already a symmetry statement: once `beta` is fixed, the only remaining
freedom is at block boundaries.

## 3. What is genuinely symmetric

The coarse bridge says that `beta` is a global quotient with deterministic drift

\[
\beta \mapsto \beta - 1 \pmod m.
\]

Inside each regular block, the transport law is therefore completely uniform:

\[
(\beta,\text{extra state}) \mapsto (\beta-1,\text{same extra state})
\qquad (\beta > 0)
\]

for any bridge written in a boundary-adapted gauge.

So the **rotatable part** is now clear:

1. the carry position is globally normalized by `beta`;
2. the countdown through the block is identical on every block;
3. the only place where any extra state can matter is the splice
   `beta = 0 -> m-1`.

This means the old “essential carry asymmetry” fork is now too strong.
There is no evidence left that the carry itself resists globalization.
The asymmetry has been pushed entirely to the **boundary update law**.

A good slogan is:

> **the dynamics are a symmetric `beta` countdown with an asymmetric boundary
> reset.**

## 4. What is not symmetric: current block type is too coarse

The saved `073` support already gives the obstruction.
The current static block type `t_n = (a_n,b_n)` does **not** determine the next
block type `t_{n+1}`.

On the checked range, the successor sets are:

- `m=5`
  - `00 -> {00,01,10,11}`
  - `01 -> {00,11}`
  - `10 -> {00}`
  - `11 -> {00}`
- `m=7,9,11`
  - `00 -> {00,01,10,11}`
  - `01 -> {00,11}`
  - `10 -> {00}`
  - `11 -> {00,01}`

So the current two-bit chain type is not a deterministic splice state.
This is exactly the `073_r1` obstruction.

### Concrete `m=5` example

In the saved `m=5` component with `source_u = 1`, the block-type cycle is

```text
00, 00, 00, 01, 11, 00, 00, 01, 00
```

and this cycle is primitive of length `9` in the saved sample summary.
So the same current type `00` occurs with several different futures, for
example

```text
00,00,00,01,11,...
```

and later

```text
00,00,01,00,00,...
```

Hence the residual exact state cannot be read off from the current block type
alone.
It must know **where that block sits in the future block word**.

This pushes the interpretation strongly away from

- “current chain parity,” or
- “current chain type only,”

and toward

- “future-word phase,” or
- “boundary memory.”

## 5. Canonical interpretation: right-congruence of the future block word

This is the key structural point.

For each boundary position `n`, define the infinite future block word

\[
\mathbf{T}_n := t_n t_{n+1} t_{n+2} \cdots
\]

and equivalently the infinite future event word

\[
\mathbf{E}_n := W(t_n)W(t_{n+1})W(t_{n+2})\cdots .
\]

Because the four block words `W_00,W_01,W_10,W_11` are distinct,
`\mathbf{T}_n` and `\mathbf{E}_n` carry the same information.

Now define the boundary equivalence relation

\[
n \sim n' \iff \mathbf{E}_n = \mathbf{E}_{n'}
\]

and write `rho_n = [n]` for the equivalence class.

This is the canonical **right-congruence / future-word state**.

### Theorem 5.1 (canonical dynamic decoration)

Fix one accessible regular component and assume the coarse `beta` clock has been
identified on it.
Then the map

\[
x \in C_n \text{ at position } \beta(x)=j
\quad\longmapsto\quad
(j,\rho_n)
\]

defines a deterministic quotient exact for current `epsilon4`.
Moreover, any deterministic quotient exact for current `epsilon4` factors
through this one.

So, on each component, the minimal exact deterministic bridge is exactly the
skew-product of the symmetric clock `beta` with the boundary right-congruence
state `rho`.

#### Proof sketch

If two states are identified by a deterministic quotient exact for current
`epsilon4`, then they must produce the same entire future `epsilon4` word,
because the quotient is deterministic and reads the current symbol exactly at
all future times. Therefore every exact deterministic quotient is finer than the
future-word equivalence above.

Conversely, if two boundary points have the same future event word, then the
same is true after deleting the first full block, so the successor on boundary
classes is well defined. After adding the same within-block offset `beta`, the
two states still have the same current symbol and the same successor future.
So `(beta,rho)` is itself a deterministic exact quotient.

Minimality is therefore immediate. ∎

This is the cleanest orbit / right-congruence interpretation requested in the
`074` handoff.

## 6. What this means conceptually

The remaining asymmetry is best read in the following equivalent ways.

### 6.1 Future-word state

`delta` is the Myhill–Nerode state of the future decorated event word.
This is the most canonical formulation.
It does not depend on any arbitrary label choice.

### 6.2 Boundary memory

`delta` is the amount of information the system must carry from one block to the
next in order to know which decorated block comes after the splice.
This is the most dynamical formulation.

### 6.3 Not a static class of the current block

The checked successor sets show that `(a,b)` alone is not enough.
So the decoration is **not** an invariant class of the current chain by itself.

### 6.4 Not a deeper phase offset

Since the entire within-block transport already factors through `beta`, the
residual asymmetry is not another hidden clock.
It is only the boundary update state riding on top of that clock.

So among the interpretations suggested in the request, the best answer is:

> **future-word state / boundary memory, not parity-like static class.**

## 7. Finite on the checked range, but size still open

On the saved checked components, the dynamic decoration is certainly finite,
because the sampled block sequences are finite periodic cycles.
The saved component lengths visible in the `073` support are:

- `m=5`: sampled component length `9`
- `m=7`: sampled component length `27`
- `m=9`: sampled component length `53`
- `m=11`: sampled component length `87`

So the right-congruence state is finite on the checked range.
That is enough to support the qualitative claim

> the remaining asymmetry is a finite dynamic decoration on top of `beta`.

But the checked material does **not** yet show a uniform small bound on the size
of that decoration, and the primitive `m=5` example already warns that the true
state is richer than the raw two-bit current type.

Therefore the main open issue is now:

> can this canonical dynamic decoration be compressed to a small uniform state,
> or does its size genuinely grow with the splice orbit?

That is a quantitative / compression question, not a basic symmetry question.

## 8. Final symmetry claim I would record

Here is the theorem-style statement I think should now replace the older
symmetry fork.

### Final symmetry claim (researcher-2 version)

Let `X` be one accessible regular component of the D5 state-level dynamics, and
assume the coarse bridge has supplied the global clock `beta`.
Let `t_n in {00,01,10,11}` be the successive decorated block types and let
`rho_n` be the right-congruence class of the future decorated event word from
boundary `n`.

Then:

1. **symmetric part:** `beta` is the canonical global clock and carry is
   rotatable;
2. **asymmetric part:** the only remaining exact state needed for current
   `epsilon4` is the dynamic boundary state `rho`;
3. **static obstruction:** no quotient using only the current chain-static type
   `(a,b)` can be deterministic globally on the checked range;
4. **canonical bridge form:** the minimal exact deterministic bridge on a
   component is of the form `(beta,rho)`;
5. **open quantitative question:** whether `rho` compresses to a small uniform
   finite state remains open.

In short:

> **the surviving asymmetry is dynamic splice memory attached to a symmetric
> global clock, not a deeper non-removable carry asymmetry.**

That is the sharpest symmetry statement I can currently justify from the saved
`071–073` material.
