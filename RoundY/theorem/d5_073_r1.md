# D5 Researcher 1 note 073

## Coarse bridge settled; splice obstruction for the naive decorated refinement

## Scope used

This note uses only the following ingredients.

- the coarse bridge theorem already established in the previous Researcher 1 note:
  the accessible union of full regular chains has a canonical global `m`-state
  quotient, written here in the `beta` gauge;
- the `071` working-note observation that, after splitting the post-carry
  `other` event into `other_1000` and `other_0010`, each full chain carries a
  two-bit early-post-carry type `(a,b)`;
- one fresh pass through the bundled frozen `047` dataset, restricted to the
  regular family, with the split recovered from `dn`:
  - `other_1000` means `dn=(1,0,0,0)`,
  - `other_0010` means `dn=(0,0,1,0)`.

I do **not** prove a full decorated exact bridge here. What I do prove is a
precise theorem-level obstruction: the obvious chain-static refinement
`(beta,a,b)` is exact on each chain but does **not** globalize as a deterministic
quotient across the accessible splice. So the post-`072` live target really is a
**splice-compatible decorated bridge**, not merely the per-chain decorated law.

A companion machine-readable summary of the checked-range transition pass is in
`_073_support.json`.

## 1. The coarse bridge is canonical and global

Let the accessible full regular chains be

```text
C_alpha = (x_{alpha,m-1} -> x_{alpha,m-2} -> ... -> x_{alpha,0}),
```

where `beta(x_{alpha,j}) = j`, the successor satisfies

```text
x_{alpha,j} -> x_{alpha,j-1}   for j>0,
```

and the splice law is

```text
x_{alpha,0} -> x_{alpha+,m-1}
```

whenever the next accessible full chain exists.

Then

```text
pi_coarse(x_{alpha,j}) = beta(x_{alpha,j}) in {0,1,...,m-1}
```

gives a deterministic global quotient on the accessible union of chains, with

```text
beta -> beta-1   for beta>0,
0 -> m-1         across the splice.
```

This is exactly the coarse bridge already isolated in the previous note. It is
just the globalized distance-to-endpoint / normalized position quotient.

So the bridge-existence problem at the carry / position level is no longer open.

---

## 2. Why the bare `m`-state bridge is too coarse for current `epsilon4`

On a single full chain, the split current-event pattern has the checked-range
form

```text
beta = m-1 : carry_jump
beta = m-2 : other_1000 if a=1, else flat
beta = m-3 : other_0010 if b=1, else flat
1 <= beta <= m-4 : flat
beta = 0   : wrap
```

for a chain-dependent pair `(a,b) in {0,1}^2`.

So the bare `beta` quotient is exact for current `epsilon4` **iff** the values of
`a` and `b` are constant on the fibers `beta=m-2` and `beta=m-3`, respectively.
Equivalently:

- any exact quotient must separate states with the same `beta=m-2` but
  different `a`;
- any exact quotient must separate states with the same `beta=m-3` but
  different `b`.

This is the precise obstruction to the bare bridge. It is not a failure of
coarse globalization. It is a failure of `epsilon4`-exactness on top of that
coarse globalization.

That matches the `072` sync: the bare `m`-state bridge exists, but it merges the
first two post-carry decorations.

## 3. The first naive decorated refinement

The obvious refinement is the **chain-static** decorated map

```text
Pi_stat(x_{alpha,beta}) = (beta, a_alpha, b_alpha),
```

where `(a_alpha,b_alpha)` is constant on the entire chain `C_alpha`.

On each fixed chain this is exact for the split current event and deterministic
for the within-chain successor.

So the real question is whether this per-chain decorated quotient glues across
splices.

### Proposition 3.1 (static-label splice criterion)

Let `delta_alpha` be any label which is constant on each full chain `C_alpha`,
and define

```text
Pi_delta(x_{alpha,beta}) = (beta, delta_alpha).
```

Then `Pi_delta` is a deterministic quotient on the accessible union of chains
**iff** there is a function `Phi` on the accessible chain labels such that

```text
delta_{alpha+} = Phi(delta_alpha)
```

for every accessible splice `C_alpha -> C_alpha+`.

#### Proof

Inside a chain, determinism is automatic because `delta_alpha` is constant and
`beta` decreases by `1`.

At the splice, the quotient state `(0,delta_alpha)` must have a unique image.
But its image is exactly `(m-1,delta_{alpha+})`. So uniqueness of the quotient
successor is equivalent to `delta_{alpha+}` being a function of `delta_alpha`.
The converse is immediate. ∎

### Corollary 3.2

The chain-static decorated object `(beta,a,b)` globalizes deterministically **iff**
the next chain type `(a_{alpha+},b_{alpha+})` is a function of the current type
`(a_alpha,b_alpha)`.

So the bridge problem has sharpened from

> “is there any global bridge?”

into

> “does the early-post-carry decoration define a splice-compatible state?”

## 4. Checked-range obstruction for the chain-static `(beta,a,b)` bridge

I extracted every regular full chain in the bundled frozen `047` data as a
length-`m` block from `carry_jump` to the following `wrap`, split the two kinds
of `other` using `dn`, and then recorded the chain type `(a,b)` and the next
chain type after the splice.

The chain-type counts reproduce the `071` working-note counts:

- `m=5`:  `00:19`, `01:5`,  `10:1`, `11:2`
- `m=7`:  `00:103`, `01:17`, `10:3`, `11:12`
- `m=9`:  `00:299`, `01:37`, `10:5`, `11:30`
- `m=11`: `00:655`, `01:65`, `10:7`, `11:56`

What matters for globalization is the **next-type map**. On the checked range,
it is not a function of the current type.

### Successor sets of the current chain type

For the chain-static refinement, the wrap state `(beta=0,a,b)` would need a
unique successor `(m-1,a+,b+)`. Instead the checked data show:

- `m=5`
  - `00 -> {00,01,10,11}`
  - `01 -> {00,11}`
  - `10 -> {00}`
  - `11 -> {00}`

- `m=7,9,11`
  - `00 -> {00,01,10,11}`
  - `01 -> {00,11}`
  - `10 -> {00}`
  - `11 -> {00,01}`

So the one-step type-Markov property fails already for the most common chain
class `00`, and it also fails for `01` on all checked moduli and for `11` once
`m>=7`.

Equivalently, the following chain-static states are nondeterministic in the
checked data:

- `(beta=0,ab=00)` on `m=5,7,9,11`
- `(beta=0,ab=01)` on `m=5,7,9,11`
- `(beta=0,ab=11)` on `m=7,9,11`

For example, on `m=7` the current chain type `00` is followed by all four
possible next types in the extracted regular traces.

### Theorem 4.1 (checked-range obstruction)

On the bundled checked range `m=5,7,9,11`, the chain-static decorated quotient

```text
Pi_stat = (beta,a,b)
```

is **not** a deterministic global quotient on the accessible union of full
regular chains.

#### Proof

By Corollary 3.2, it would be enough that the next chain type were a function of
`(a,b)`. The transition summary above shows that this fails. Therefore
`Pi_stat` is not deterministic at the splice. ∎

## 5. What the exact bridge target now has to be

The `073` picture is therefore:

1. **coarse bridge:** settled and global;
2. **bare exact bridge for `epsilon4`:** impossible, because `beta` alone merges
   the `m-2` and `m-3` decorations;
3. **naive chain-static decorated bridge `(beta,a,b)`:** exact on each chain,
   but not splice-compatible as a deterministic global quotient on the checked
   range;
4. **live target:** a splice-compatible decorated bridge.

So the exact bridge we now want cannot just remember the current chain’s static
post-carry type. It must remember the decoration in a **boundary-aligned** way
that already determines the next chain type at the splice.

A clean way to say this is:

> the exact global bridge must encode a right-congruence class of the future
> decorated-chain word, not merely the current chain’s static two-bit type.

This is the precise theorem-level obstruction I can currently justify.

It also explains how to interpret the `072` sync paragraph without contradiction:
if a small decorated bridge of the form `(beta,a,b)` is truly supported on the
checked range, then the bits `a,b` cannot be read as the **static type of the
current chain**. They must be a splice-compatible / dynamically updated
interpretation of those decorations.

---

## 6. Bottom line for Researcher 1

I would now state the Researcher 1 result as follows.

### Coarse bridge theorem

The per-chain exact marked-chain quotients do glue into one global coarse
quotient on the accessible union of full regular chains.

### Precise obstruction for the refined bridge

The first naive refinement obtained by attaching the chain’s two early
post-carry decoration bits `(a,b)` to `beta` is exact on each chain but fails to
globalize deterministically across the splice on the checked range.

### Consequence

The live exact-globalization problem is no longer bridge existence. It is the
construction of a **splice-compatible decorated exact bridge**: a refinement of
`beta` that separates the early post-carry events and carries enough boundary
information to predict the next chain’s decoration.

That is a precise obstruction about globalization of the per-chain exact
quotients themselves, and it keeps the statement on the exact quotient rather
than on broad local-rule families.
