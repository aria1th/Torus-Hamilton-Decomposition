## 1. The splice step is explicit in the checked data

For each full regular chain, keep the chain label

- `q` = the normalized carry-slice coordinate from the earlier bridge work,
- `sigma = source_u + w (mod m)`.

On the observed full-chain splices, the next chain is a **deterministic**
function of `(q,sigma)`.

The checked rule is:

```text
(q, sigma)  ->  (q + 1 mod m, sigma + 1_{q = m-1} mod m).
```

This matched every observed splice on `m = 5,7,9,11`.

Coverage:

- `m = 7,9,11`: the observed full-chain state space is the full `m^2` grid of
  `(q,sigma)` pairs;
- `m = 5`: the observed full-chain state space has `23` pairs, missing only
  `(q,sigma) = (4,0), (4,2)` in the frozen rows.

So the dynamic part missing from the static `073` bridge is not an opaque large
controller state. On the checked range it is a concrete splice phase
`sigma`.

## 2. Supported dynamic bridge

Attach `(q,sigma)` to the coarse clock `beta`.

The candidate quotient is

```text
(m, beta, q, sigma).
```

On the checked frozen rows, current `epsilon4` is given exactly by

```text
beta = m-1 : carry_jump
beta = 0   : wrap
beta = m-2 : other iff q = m-1
beta = m-3 : other iff (q + sigma = 1 mod m) or (q = m-1 and sigma != 1)
otherwise  : flat
```

and the short-corner detector is exactly

```text
short_corner = 1 iff beta = m-2, q != m-1, and q + sigma = 1 mod m.
```

Direct row check status on the observed full-chain union:

| modulus | observed `(beta,q,sigma)` states | `epsilon4` exact | deterministic successor | short-corner exact |
|---|---:|---|---|---|
| 5 | 115 | yes | yes | yes |
| 7 | 343 | yes | yes | yes |
| 9 | 729 | yes | yes | yes |
| 11 | 1331 | yes | yes | yes |

So this is a genuine **splice-compatible decorated bridge** on the checked
range.

## 3. Candidate quotients tested

I tested the following focused candidate family on the full regular-chain union
with observed splices.

| candidate quotient | current `epsilon4` exact? | deterministic? | short-corner exact? | first failure |
|---|---|---|---|---|
| `(beta)` | no | yes | no | at `m=5`, `beta=3` mixes `flat` and `other` |
| `(beta,a,b)` | yes | no | yes | at `m=5`, `(beta,a,b)=(0,0,0)` wraps to all four next `(a,b)` types |
| `(beta,q)` | no | yes | no | at `m=5`, `(beta,q)=(2,0)` mixes `flat` and `other` across different `sigma` |
| `(beta,sigma)` | no | no | no | at `m=5`, `(beta,sigma)=(3,3)` mixes `flat` and `other`; `(0,3)` has two next states |
| `(beta,q,b)` | yes | no | yes | at `m=5`, `(beta,q,b)=(0,0,0)` wraps to next `(q,b)=(1,0)` and `(1,1)` |
| `(beta,q+sigma)` | no | no | no | too coarse for both event exactness and splice prediction |
| `(beta,q,sigma)` | yes | yes | yes | survives all checked rows |

Equivalent exact reparameterization:

- `(beta,q,r)` with `r = q + sigma (mod m)` is also exact and deterministic,
  since `(q,r)` is bijective with `(q,sigma)`.

### First concrete counterexamples

1. **Static decorated bridge fails determinism.**  
   At `m=5`, the state key `(beta,a,b)=(0,0,0)` occurs on several wrap states,
   and the observed next decorated carry states include

   ```text
   (4,0,0), (4,0,1), (4,1,0), (4,1,1).
   ```

   So chain-static decoration does not globalize.

2. **`(beta,q)` is still too coarse.**  
   At `m=5`, key `(beta,q)=(2,0)` contains

   - a row with `sigma=3`, `epsilon4=flat`,
   - a row with `sigma=1`, `epsilon4=other`.

   So `q` alone does not encode the second post-carry event.

3. **`(beta,q,b)` is exact but still not deterministic.**  
   At `m=5`, key `(beta,q,b)=(0,0,0)` wraps to next keys

   ```text
   (4,1,0) and (4,1,1),
   ```

   depending on `sigma`.

These are the relevant obstruction witnesses in the targeted small family.

## 4. Right-congruence / future-word computation

After identifying the supported splice law, I computed Moore / right-congruence
classes on the symbolic automaton generated by

```text
(q, sigma) -> (q+1, sigma + 1_{q=m-1})
```

with current-chain output `(a,b)` and with full state output given by current
`epsilon4`.

### Chain-level result

For `m = 7,9,11`, every one of the `m^2` chain states `(q,sigma)` is in its own
Moore class. So there is **no smaller deterministic future-equivalent quotient**
of the supported dynamic chain decoration on those checked moduli.

For `m=5`, the same is true on the induced full symbolic `5 x 5` completion,
but that goes one step beyond the directly observed `23` chain states because two
pairs are absent from the frozen rows.

### Full state result

For the symbolic full bridge state `(beta,q,sigma)`, the Moore class count is

- `125` for `m=5`,
- `343` for `m=7`,
- `729` for `m=9`,
- `1331` for `m=11`,

i.e. **all symbolic states remain distinct**.

Interpretation:

> within the supported symbolic splice model, the smallest deterministic exact
> bridge visible to the checked data is the full `(beta,q,sigma)` bridge.

This is stronger than merely saying that `(beta,q,sigma)` works. It says the
data do **not** support any smaller factor of that supported symbolic bridge on
the checked range.

## 5. What this means for the frontier

The live compute answer is now sharper.

- The missing splice datum is a **dynamic phase** `sigma`, not an unexplained
  large controller.
- The first splice-compatible decorated bridge supported by the checked data is

  ```text
  (beta,q,sigma).
  ```

- The older static decorated bridge `(beta,a,b)` should now be viewed as a
  per-chain shadow of that dynamic bridge, with

  ```text
  a = 1 iff q = m-1
  b = 1 iff (q + sigma = 1 mod m) or (q = m-1 and sigma != 1).
  ```

- Within the supported symbolic model, no smaller deterministic exact bridge is
  visible on the checked range.

So the active target is no longer “can the splice be modeled at all?” but
rather:

> can the theorem side identify and justify this dynamic phase intrinsically,
> or prove an equivalent smaller parametrization if one exists outside the
> currently supported symbolic model?
