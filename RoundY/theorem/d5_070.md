Using the `069` handoff as the base, I think the right first exact object is indeed a **marked length-(m) chain**, not a full cycle. The note now explicitly treats exact reduction as the first live focus, says the safer first object is a marked length-(m) chain extracted from regular carry-jump slices, and says the cycle should be treated as a later strengthening. It also records current support from full regenerated active rows through (m=21) and branch-local support through (m=41).  

The reason is simple: the phase-corner theorem already gives you a forward (m)-step object for free, because (\kappa'=\kappa+1\pmod m); what it does **not** give for free is the return identification needed to close that object into a cycle. The theorem-side machine is “odometer with one corner,” but exact reduction should first use only the part that is already forced: a deterministic (m)-window cut from the active branch.  

## 1. The right first exact object

Let (T) be the active-branch successor. Assume the phase-corner theorem on the active best-seed branch:

[
\kappa'=\kappa+1 \pmod m,
]

and the current event (\epsilon_4) is given by the small scheduler, with one short corner at ((\kappa,s)=(2,2)). 

Choose an anchor (a) on a regular carry-jump slice, and define

[
x_i:=T^i a,\qquad 0\le i\le m-1.
]

Set

[
C_m(a):=(x_0,x_1,\dots,x_{m-1}).
]

I would call this the **marked exact chain candidate**. Its marks are:

[
M_c:={,x_i : c(x_i)=1,},
\qquad
x_\star:=\text{the unique short-corner state, if it lies on the chain}.
]

I am not assuming yet that (M_c) is singleton, only that it is a well-defined marked subset. The handoff’s exact-reduction target is phrased this way too: first extract the marked chain/cycle, then identify the marked carry state(s), then determine the induced quotient and whether it is exact for (\epsilon_4). 

## 2. Why the chain is already exact

### Theorem A

Under the phase-corner theorem, (C_m(a)) is a deterministic length-(m) chain.

More precisely:

1. (x_0,\dots,x_{m-1}) are pairwise distinct.
2. The successor law is exact:
   [
   T(x_i)=x_{i+1}\qquad(0\le i<m-1).
   ]
3. Each residue of (\kappa) appears exactly once on the chain.
4. Hence the chain contains exactly one wrap state, exactly one carry-jump state, and at most one short-corner state.

### Proof

Because (\kappa'=\kappa+1\pmod m),

[
\kappa(x_i)=\kappa(a)+i \pmod m.
]

If (x_i=x_j) for some (0\le i<j<m), then (\kappa(x_i)=\kappa(x_j)), so

[
i\equiv j \pmod m.
]

But (0<j-i<m), impossible. So the (x_i) are distinct. The successor statement is by construction. Since the residues (\kappa(a),\kappa(a)+1,\dots,\kappa(a)+m-1) are all distinct mod (m), every (\kappa)-residue appears exactly once. The scheduler has wrap only at one phase, carry-jump only at one phase, and the short corner only at ((\kappa,s)=(2,2)), so those marks occur at most once each on the chain. ∎

This is the main exact-reduction point: the theorem package already proves a forward (m)-window. No return assumption is needed.

## 3. Why the cycle should be treated as a promotion

Let

[
x_m:=T x_{m-1}.
]

The phase law does **not** imply (x_m=x_0), nor even that (x_m) should be identified with (x_0) in the intended quotient. That is extra information.

### Proposition B

The exact chain (C_m(a)) promotes to a quotient cycle only if the intended quotient identifies (x_m) with (x_0).

So the correct logical order is:

1. prove the chain;
2. compute the induced quotient on the chain;
3. test whether terminal successor closes to the initial class.

That is exactly the chain-vs-cycle nuance the recent handoffs emphasize.  

## 4. The induced exact quotient of the intended class

Now fix the intended local/admissible class. Let

[
\mathcal O:C_m(a)\to\Lambda
]

be the current datum available to that class on the chain. This could be the grouped current datum the class actually sees; the point is that (\mathcal O) is the base observable of the intended class.

For each (i), define the suffix word

[
W_i:=\big(\mathcal O(x_i),\mathcal O(x_{i+1}),\dots,\mathcal O(x_{m-1})\big).
]

Define an equivalence relation by

[
x_i \sim_{\mathcal O} x_j
\iff
W_i=W_j.
]

Let

[
Q_{\mathcal O}(C_m):=C_m(a)/{\sim_{\mathcal O}}.
]

This is the exact quotient I would use. It is the finite-chain analogue of the minimal deterministic quotient.

### Theorem C

(Q_{\mathcal O}(C_m)) is the smallest quotient induced by (\mathcal O) on which the chain successor is deterministic.

### Proof

If (W_i=W_j), then in particular (\mathcal O(x_i)=\mathcal O(x_j)), and after deleting the first symbol from each word we still get equality, so

[
W_{i+1}=W_{j+1}
]

whenever both sides are defined. Therefore successor descends to the quotient.

Now let (q:C_m(a)\to Q) be any quotient such that:

1. (\mathcal O) factors through (q), and
2. successor is deterministic on (Q).

If (q(x_i)=q(x_j)), then by determinism

[
q(x_{i+t})=q(x_{j+t})
]

for every (t) for which both are defined. Since (\mathcal O) factors through (q), we get

[
\mathcal O(x_{i+t})=\mathcal O(x_{j+t})
]

for all such (t). Hence (W_i=W_j), so (x_i\sim_{\mathcal O}x_j).

Thus every deterministic quotient induced by (\mathcal O) refines (\sim_{\mathcal O}). So (Q_{\mathcal O}(C_m)) is the smallest one. ∎

This is the exact quotient induced by the intended class in the cleanest possible sense.

## 5. Why this is the right reduction for the intended class

If the intended class allows hidden carrier states (s_n\in S) updated by current observable data,

[
s_{n+1}=U(\mathcal O(x_n),s_n),
]

then every realization over that class sits over the base quotient (Q_{\mathcal O}(C_m)). Concretely, on the actual run one can map

[
x_i \mapsto \big([x_i]*{\sim*{\mathcal O}},, s_i\big)\in Q_{\mathcal O}(C_m)\times S.
]

So the right reduction is:

* first reduce the chain to its exact observable quotient (Q_{\mathcal O}(C_m)),
* then study whether any carrier over that base can make (\epsilon_4), and later (\beta), exact.

That matches the current project split: exact reduction first, realization second. 

## 6. Exactness criteria you can test immediately

### Corollary D: (\epsilon_4)-exactness criterion

The induced quotient (Q_{\mathcal O}(C_m)) is exact for the current event (\epsilon_4) iff (\epsilon_4) is constant on every (\sim_{\mathcal O})-class.

Equivalently:

[
W_i=W_j \Longrightarrow \epsilon_4(x_i)=\epsilon_4(x_j).
]

This is the clean first diagnostic.

### Corollary E: cycle-promotion criterion

Extend one step to (x_m=T x_{m-1}). Then the quotient closes to a cycle iff (x_m) has the same future observable behavior as (x_0), i.e. iff the induced quotient identifies terminal successor with the initial class.

So the exact-reduction compute should test:

1. extract (x_0,\dots,x_m),
2. build (W_i),
3. form (Q_{\mathcal O}(C_m)),
4. test (\epsilon_4)-exactness,
5. test whether (x_m) closes back to (x_0).

That directly answers the handoff’s questions: chain or cycle, smallest accessible quotient, exactness for (\epsilon_4), and preservation of the short corner. 

## 7. What this gives you already

So I think the formal package for the exact-reduction stage should be:

**Reduction Theorem 1 (marked-chain extraction).**
From the phase-corner theorem, every regular carry-jump slice yields a deterministic marked length-(m) chain.

**Reduction Theorem 2 (minimal induced quotient).**
For any intended current observable (\mathcal O), the suffix-word quotient (Q_{\mathcal O}(C_m)) is the smallest deterministic quotient induced by that class on the chain.

**Reduction Corollary 3 (cycle promotion).**
A full cycle is a promotion statement:
it holds exactly when terminal successor closes to the initial class in (Q_{\mathcal O}(C_m)).

**Reduction Corollary 4 ((\epsilon_4)-exactness).**
The quotient is exact for (\epsilon_4) exactly when (\epsilon_4) is constant on the quotient classes.

That is enough to move the live frontier from vague controller search to a finite exact-reduction problem.

## 8. What is proved now, and what is still only validated

What I think is genuinely proved in shape now:

* chain before cycle is the right logical order;
* once the phase-corner theorem is granted, the marked length-(m) chain is immediate;
* the exact quotient induced by the intended class is the future-observable quotient above.

What is still only validated/support-level, not closed from the handoff alone:

* the existence and clean extraction of the intended regular carry-jump anchors for all odd (m);
* whether the observable quotient actually closes to a cycle;
* whether it is exact for (\epsilon_4);
* whether it already preserves the unique short corner.

Those are exactly the next exact-reduction checks the `069` handoff says to do. 

The upshot is: yes, I think the right exact object is **first** a marked length-(m) chain, and the right quotient is the **minimal deterministic quotient induced by the intended observable class** on that chain. A full cycle is not the starting point; it is a promotion theorem.
