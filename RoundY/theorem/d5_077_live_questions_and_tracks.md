# D5 Live Questions And Tracks 077

This note isolates the current live D5 questions after the accepted `076`
round and separates them into the three active work streams.

It is meant to be the shortest statement of:

- what the real yes/no question is,
- what the equivalent formulations are,
- what each track should do,
- and what should no longer be reopened.

## 1. Main live question

The current D5 question is now:

> does raw global `(beta,delta)` work across the full accessible union without a
> component tag?

The equivalent formulation is:

> can the same realized `delta` occur in different accessible components with
> different padded future current-event words?

If the answer is **no**, then raw global `(beta,delta)` is the correct exact
bridge.

If the answer is **yes**, then the correct final theorem object stays abstract
as `(beta,rho)` globally, with `(beta,delta)` valid only componentwise.

So the live frontier is no longer “find the bridge.” It is:

- **global raw `delta`**, or
- **abstract `rho` globally plus concrete `delta` only componentwise**.

## 2. Current accepted picture

These points should be treated as settled context.

- The structural theorem package through the phase-corner machine is no longer
  the main frontier.
- The per-chain marked-chain rule is no longer the main frontier.
- The coarse bridge `beta` is no longer the main frontier.
- The abstract bridge `(beta,rho)` is accepted as the safest theorem object.
- The abstract realization theorem is accepted in near-final form.
- The concrete odometer bridge `(beta,q,sigma)` / `(beta,delta)` is accepted
  componentwise as the strongest current model.

So the remaining uncertainty is **globalization**, not local rule discovery.

## 3. Track A: global component structure

### Goal

Understand the actual splice-connected structure of the accessible regular union.

### Main question

What are the true accessible splice-connected components?

### Why this matters

If the full accessible union is already one suitable total component, then raw
global `(beta,delta)` becomes much more plausible.

If the union really splits into several components, then the exact question is
whether those components can realize the same `delta` with different future
words.

### Useful outputs

- a theorem or convincing structural argument describing the accessible
  components;
- a proof that there is only one relevant total component, or
- a proof that several components exist and how they intersect the concrete
  `delta` labeling.

## 4. Track B: globalization criterion / `rho` dependence

### Goal

Decide whether `rho` depends only on `delta`, or on `(component,delta)`.

### Main question

For a realized boundary label `delta`, is the padded future current-event word
determined by `delta` alone?

### Equivalent theorem target

Prove or disprove:

`rho = rho(delta)`

on the full accessible union.

### Why this matters

This is the exact bridge globalization criterion.

- If true, raw global `(beta,delta)` is the final exact bridge.
- If false, the final theorem must keep `(beta,rho)` abstract globally.

### Useful outputs

- a proof that repeated `delta` values always carry the same padded future word;
- or a proof/counterexample that some repeated `delta` values carry different
  padded future words.

## 5. Track C: compute support to connect the D5 cases

### Goal

Support or break the globalization criterion using only targeted compute.

### What to compute

- repeated-`delta` ambiguity on the true accessible unions, not just on proxy
  unions;
- accessible-image closure on larger odd moduli;
- larger-odd-`m` support for the splice law and current-event readout only when
  it directly supports the globalization question;
- actual component decomposition evidence if structural proof is not yet
  available.

### What not to compute

- no broad controller search;
- no reopening old tiny-catalog scans;
- no new speculative bridge families unless they directly test the
  globalization criterion.

### Useful outputs

- first real repeated-`delta` ambiguity witness on an actual accessible union,
  if it exists;
- or strong evidence that all repeated realized `delta` values have identical
  future words.

## 6. Practical order

The recommended order is:

1. keep the theorem language safe at the abstract `(beta,rho)` level;
2. work on global component structure in parallel with the `rho(delta)` question;
3. use compute only to test the exact globalization criterion;
4. only after that decide the final theorem form:
   - raw global `(beta,delta)`, or
   - abstract global `(beta,rho)` with componentwise concrete charts.

## 7. Bottom line

The branch is now close in a very specific sense:

- the local rule is essentially identified,
- the componentwise bridge is essentially identified,
- and the only remaining sharp question is whether that concrete bridge
  globalizes without a component tag.

That is the current D5 frontier.
