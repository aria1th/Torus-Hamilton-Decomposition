The cleanest current statement is a split one:

**The offset is removable on the larger exact theorem-side state space, but it is not removable in the currently checked admissible/local picture.** So carry is not absolutely asymmetric, but it is still **admissibly asymmetric across chains** in the precise sense relevant to the present bridge problem.

1. **There is a global exact normalization once the theorem-side clock is allowed.**
   On the carry-jump presentation of the regular marked chains, the checked theorem-side formula is
   [
   q \equiv 1-s-v-\mathrm{layer}\pmod m,
   ]
   so the chain-dependent affine shift disappears and carry becomes the common mark
   [
   c = 1_{q=m-1}.
   ]
   More globally, on the full nonterminal active branch,
   [
   q \equiv -\beta - s - v - \mathrm{layer}\pmod m,
   ]
   and the regenerated full-row checks say ((B,\beta)) is exact for (q,c,\epsilon_4,\tau,\text{next_}\tau,\text{next_}B). In that sense, carry **is** rotatable once the global clock (\beta) has descended. [055 boundary-reset support](../../artifacts/d5_boundary_reset_proof_support_055/README.md), [068B exact-reduction support](../../artifacts/d5_exact_reduction_support_068b/README.md)

2. **But the currently extracted admissible chain quotient does not remove the offset.**
   Per chain, the checked admissible coordinate is exactly the affine gauge
   [
   s = q+\sigma,\qquad \sigma = \text{source_u}+w \pmod m.
   ]
   So each fixed chain is an exact (m)-state counter, but with a chain-dependent endpoint (s=\sigma-1). Two direct obstructions follow.

* (s) is **not** globally carry-exact.
  Take the endpoint of chain (C_\sigma): it has (q=m-1), hence (c=1), and (s=\sigma-1).
  On any different chain (C_\tau), there is a unique state with the same (s=\sigma-1), namely (q=\sigma-1-\tau), and because (\tau\neq\sigma) that state is not the endpoint, so (c=0).
  Thus carry cannot factor through (s) on the union of chains.

* Even ((s,\text{endpoint-bit})) is **not** deterministic.
  Write the endpoint/interior mark as `next_dn` (flat on interior, `other_1000` at the endpoint).
  Take the state (x\in C_\sigma) with (q=m-2). Then ((s,\text{next_dn})(x)=(\sigma-2,\text{flat})), but its successor is the endpoint.
  Choose a different chain (C_\tau) with (\tau\notin{\sigma,\sigma-1}), and let (y\in C_\tau) be the unique state with the same (s=\sigma-2). Then (y) also has `next_dn = flat`, but its successor is still interior, not the endpoint.
  So successor does not descend through ((s,\text{next_dn})).

That is a genuine obstruction: any normalization that only uses the verified per-chain admissible counter (s) and the exact endpoint mark cannot globalize the chains.    The exhaustive 69,994-family search on the longer spliced source chains finds that the smallest exact deterministic quotient has full size
   [
   m(m-3),
   ]
   not size (m). On the checked moduli this is (10,28,54,88) for (m=5,7,9,11). So within the entire checked admissible catalog, no family removes the offset and rotates the chains to one common (m)-state carry counter. So the theorem/counterexample split I would record for **Researcher 2** is:

> **Supported symmetry:** on the theorem-side exact state space, there is a global (q)-normalization, hence carry is rotatable.
> **Admissible obstruction:** on the currently checked admissible/local quotient, the offset (\sigma) is not globally removable; the per-chain quotient is only the affine (s)-gauge, and exhaustive checked admissible families do not collapse the spliced chains to one (m)-state rotation.

That pins down the two pictures sharply: the real bridge problem is not whether a global (q)-coordinate exists somewhere—it does—but whether the extra global clock data needed for that normalization can descend to the intended admissible quotient.
