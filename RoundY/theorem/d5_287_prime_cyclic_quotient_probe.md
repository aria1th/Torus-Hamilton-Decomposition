# D5 287 Prime Cyclic Quotient Probe

This note records one focused experimental check on the proposed
prime-dimension `C_p` quotient / voltage viewpoint.

The question is not whether quotient or monodromy language is ever useful.
The question is narrower:

> in the already closed `d=3` Route-E package and in the current `d=5`
> backbone / Route-E package, does the fixed-color dynamics already descend
> cleanly from the **full torus** to the naive quotient by cyclic coordinate
> rotation?

The answer from the present probe is:

- **no** at the full-torus level;
- **yes, in a different form**, only after the existing return-ladder
  compression has exposed the right reduced object.

## 1. D3 control case

The D3 control is the closed even Route-E package from the arXiv-ready
ancillary code. The probe checks three different levels.

### 1.1 Full-torus cyclic covariance fails

For `m = 6, 8, 10`, the simple covariance ansatz

`sigma ∘ F_c = F_{c+s} ∘ sigma`

fails for every candidate color shift `s`.

So even in a completely closed low-dimensional package, the full family is not
organized by naive cyclic coordinate transport.

### 1.2 Full-torus fixed-color quotient descent fails

The same moduli show that, on free `C_3` orbits, the fixed-color maps do not
descend deterministically to the naive cyclic quotient.

The first witness already appears on the orbit of `(0,0,1)`: different points
in that orbit can map to different orbit classes. So the quotient is not the
right theorem object at the raw full-map level.

### 1.3 The right reduced objects are return-ladder objects

By contrast, the existing Route-E return-ladder data are completely clean.

For `m = 6, 8, 10`:

- each displayed return map on `P0` is one `m^2`-cycle;
- each transversal first-return map is one `m`-cycle;
- each transversal return-time sum is exactly `m^2`.

So the D3 lesson is not “quotients are useless.” It is:

> the useful structure appears only after the paper's existing return-map
> compression, not on the full torus modulo raw coordinate rotation.

## 2. D5 test case

The D5 probe uses the current `Sel*` graph-side package, especially the closed
color-`3` Route-E branch.

### 2.1 Full-torus cyclic covariance fails

For `m = 5, 7`, the same simple covariance ansatz fails for every candidate
color shift on the full `Sel*` family.

So the natural prime-dimension idea

- rotate coordinates,
- relabel colors cyclically,
- then descend to a quotient

does not hold even as a full-family symmetry of the present concrete package.

### 2.2 Full-torus fixed-color quotient descent fails

Again for `m = 5, 7`, every full fixed-color map has nondeterministic free
`C_5` orbit images on the naive cyclic quotient.

So the full-torus quotient-first program is not even correct at the checked
concrete level.

### 2.3 Even the first return on `P0` is still too early

For the closed color-`3` branch, the probe next checks the first return

`R_3^* = (F_3^*)^m | P0`.

For `m = 5, 7, 9, 11`, this still does **not** descend deterministically to
the naive cyclic `C_5` quotient on free orbits.

So in D5 the right reduced object is not yet visible even after one descent to
`P0`.

### 2.4 The useful structure appears one step later

The same checked moduli show that the actual color-`3` return ladder is
completely rigid at the section-model level.

Specifically:

- the actual `P0` first return and actual section return formulas are exact;
- the explicit section model is one `m^3`-cycle for checked `m = 9, 11`;
- the explicit row model is one `m^2`-cycle for checked `m = 9, 11`.

So the D5 message matches the D3 control:

> the simple cyclic/clock structure only becomes visible after the existing
> return-ladder compression has already exposed the correct reduced section or
> row model.

## 3. Methodological consequence

This probe materially changes how the prime-dimension generalization idea
should be read.

The safe current reading is:

- a naive full-torus `C_p` quotient is **not** the right primary theorem
  object;
- a naive quotient-first search for a base motif on `T / C_p` is therefore not
  supported by the present D3 and D5 data;
- cyclic-quotient, monodromy, or voltage language may still be useful, but
  only as a **late-stage description** of an already extracted reduced return
  model.

So the current generalization direction should remain the one already suggested
by
[d5_282_backbone_defect_compression_methodology.md](/data/angel/repos/modulation-guidance/Torus-Hamilton-Decomposition/RoundY/theorem/d5_282_backbone_defect_compression_methodology.md):

- freeze exact backbone colors first;
- identify raw residual obstructions;
- descend by return ladders until exact clocks appear;
- split by resonance;
- localize defects to thin supports;
- and only then ask whether a finite quotient / voltage / gate description is
  available.

## 4. What the later Opus sketch gets right and wrong

The later note
[idea_to_general_opus_4.6_2.md](/data/angel/repos/modulation-guidance/Torus-Hamilton-Decomposition/tmp/idea_to_general_opus_4.6_2.md)
is substantially closer to the present D5 evidence than the earlier
quotient-first sketch.

Its main improvement is that it no longer treats quotient or voltage language
as the top-level starting point. Instead, it tries to reinterpret the actual
`d=5` program in bottom-up terms:

- exact backbone first,
- return-map descent next,
- resonance as a local drift obstruction,
- support-widening surgery only after the obstruction is isolated.

That much is strongly aligned with the current D5 branch.

### 4.1 Parts that are now strongly supported

These parts are not merely plausible. They are already close to the present
working D5 picture.

- **Backbone first**. The note is right that the real D5 progress came from
  freezing colors `3,4` first, not from solving a quotient graph in advance.
- **Return-map descent as the real reduction engine**. This is exactly how the
  closed color-`3` and color-`4` branches behave.
- **Resonance should be read as vanishing local drift / voltage**. This is a
  good reinterpretation of the present `3 | m` residual obstruction. The probe
  does not prove the final voltage language, but it is consistent with the
  actual residual picture.
- **Support widening should be tied to exact drift laws**. This matches the
  current width / collar / breaker language far better than the earlier global
  quotient-first program.

So the Opus note is correct to shift the general philosophy from

- quotient-first

to

- backbone-first plus return-ladder compression.

### 4.2 Parts that are consistent but still unproved

Some claims in the Opus note are good program targets, but the current D3/D5
experiments do not yet prove them.

- **`O(1)` defect slices for general prime `p`**. This is a good and important
  gate, but at present it is still only a programmatic conjecture.
- **A general “easy colors first” backbone split for prime `p`**. Again, this
  is methodologically reasonable, but not yet supported beyond low-dimensional
  case evidence.
- **Piecewise-affine section returns with branch count polynomial in `m` for
  general `p`**. This is consistent with the current D5 experience, but still
  speculative at general prime dimension.
- **A D7 Phase-0 / Phase-1 reconnaissance based on slice failures and easy
  backbone colors**. This is probably the right next experimental program, but
  it remains exactly that: a program, not yet a theorem-guided closure route.

These are not contradicted by the present probe. They are simply not yet
verified.

### 4.3 Parts that remain too strong

Even after the Opus correction, a few phrases are still stronger than the
current evidence justifies.

- The note says that after the backbone is fixed, “quotient이 자동으로
  따라왔다.” That is too strong as written. The current experiments support a
  weaker statement:
  the **right reduced object becomes visible only after the return ladder has
  already been built**. A useful quotient may appear later, but it does not
  come for free immediately from fixing a backbone.
- The note also treats the resonance obstruction as if the correct voltage
  space were already known. The present evidence only supports the softer
  claim that the residual obstruction behaves *as if* a local drift/voltage
  vanishes on the reduced return object. The exact general prime formalism is
  still to be identified.
- The proposed width-expansion law for general `p` is a good heuristic, but it
  should not yet be read as a stable template. In D5, the successful local
  widening laws are very specific to the extracted residual geometry.

So the Opus note is a good compression of the present methodology, but it is
still best read as:

- a **research program for general prime `p`**, not
- a theorem blueprint already validated beyond D3/D5.

### 4.4 Net assessment

Relative to the earlier Gemini-style quotient-first note, the Opus note is a
major improvement.

The present best assessment is:

- the **front half** of the Opus program is strongly supported:
  backbone first, return ladders, resonance detection, defect-supported
  surgery;
- the **back half** is still open:
  general prime `p` should not yet be expected to inherit the same clean
  quotient or transducer package without a new casebook / reconnaissance stage.

So the safest current formulation is:

> the Opus note is promising as a methodological redesign of the general-prime
> program, but only after one weakens its quotient claims and treats the
> `O(1)`-defect / explicit-transducer expectations as open gates rather than
> established facts.

## 5. Bottom line

The current probe does **not** say that cyclic quotient ideas are wrong.

It says something more precise:

> they appear to be the wrong **starting point**, but a plausible
> **ending language** once the correct reduced return object has already been
> identified.

That is the present best correction to the earlier quotient-first prime-`p`
sketch.
